Linux 命令

Linux 目录结构

  • 为树型结构
  • 只有一个根目录
  • 层级关系用 / 表示
  • 出现在开头的根目录用 / 表示

Linux命令入门

Linux命令基础
  • 通用格式 command [- options] [parameter]
    ls命令入门
  • ls [-a -l -h] [Linux 路径]
    参数
  • -a 即all，列出全部文件（包含隐藏的文件/文件夹）
  • -l 以列表（竖向排列）的形式展示内容，并展示更多信息
  • -h 需要和-l配合使用，显示文件大小
    选项
  • 路径显示
cd-pwd命令
  • cd [Linux 路径]
  • pwd 输出当前所在根目录
相对路径和绝对路径
  • 绝对路径：以根目录为起点，以/开头
  • 相对路径：以当前目录为起点
  • 特殊路径符：
    • . 表示当前目录，比如cd./Desktop 与cd Desktop一致
    • .. 表示上一级目录
    • ~ 表示HOME目录，cd~/Desktop 表示切换到HOME的Desktop目录
mkdir 命令
  • mkdir [-p] Linux 路径 （创建新文件夹）
  • -p 表示自动创建不存在的目录
文件命操作命令
  • touch Linux路径 （创建新文件）
  • cat Linux路径 （查看文件内容）
  • more Linux路径 （查看文件内容，空格翻页，B键上一页，q键退出）

  • cp [-r] 参数1 参数2

    • -r 表示递归（复制文件夹时使用）
    • 参数1，源文件
    • 参数2，目标文件

  • mv 参数1 参数2

    • 参数1，源文件
    • 参数2，目标文件 （文件不存在，则有改名效果）
  • rmdir 文件夹 删除空文件夹
  • rm [- -f] 参数1 参数2 … 参数n
    • -r 用于删除文件夹
    • -f 强制删除
    • 支持通配符
      • test* 表示匹配任何以test开头的内容
      • *test 表示匹配任何以test结尾的内容、
      • test\表示匹配任何包含test的内容
which命令
  • which 要查找的命令
find命令
  • find 起始路径 -name “被查找的文件名”
    • -name 按文件名搜索
    • -size 按文件大小搜索
      • + - 表示大于和小于
      • n 代表大小数字
      • k M G 注意大小写，表示大小
      • -type 按文件类型搜索
      • -maxdepth 指定搜索深度
      • -exec ls -ld {} \; exec 将find的结果集执行某一指定命令
      • -xargs ls -d 将find搜索的结果集执行某一指定命令。当结果集数量过大时，可以分片映射
      • -ok 交互式方式
grep命令
  • grep [-n] 关键字 文件路径
    • -n 表示在结果中显示匹配的行的行号
    • 参数 文件路径，可作为管道符的输入
wc命令
  • wc [-c -m -l -w] 文件路径
    • 不带选项默认统计：行数、单词数、字节数、字符数等
    • -c 字节数
    • -m 字符数
    • -l 行数
    • -w 单词数
    • 参数 被统计的文件路径，可作为管道符的输入
管道符

echo命令
  • echo 输出的内容

重定向符

Linux用户和权限

root用户

  • Linux系统的超级管理员用户是：root用户

  • su [-] [用户名]

      • 表示切换后加载环境变量，建议带上
    • 用户可以省略，省略默认切换到root

  • sudo 其他命令

    • 需要以root用户执行visudo命令，增加配置
    • 增加配置项如下 baiy ALL=(ALL) NOPASSWD:ALL
    • 并以wq保存

用户、用户组

  • groupadd 用户组名
  • groupdel 用户组名
  • useradd [-g -d] 用户名
    • -g 指定用户的组 若不指定则会创建同名组并自动加入
    • -d 指定用户HOME路径，不指定，HOME目录默认在: /home/用户名
  • userdel [-r] 用户名
    • -r 删除用户的HOME目录，不使用-r，删除用户时，HOME仍保留
  • usermod
    • -aG 用户组 用户名，将指定用户加入指定用户组
  • id [用户名]
    • 不加则默认查看root用户
  • getent passwd
  • getent group

查看权限控制

  • rwx 分别代表 读、写、执行权限
  • 前10个字符分别代表文件类型、所属用户权限、所属用户组权限、其它用户权限

动态库

制作
  1. 将.c 文件生成 .o （生成与位置无关的代码 -fPIC）
  2. 使用gcc -shared -o lib库名.so 文件名.o

  3. 编译可执行程序时指定所使用的动态库

    • -l 指定库名 （去掉前缀lib）
    • -L 指定库路径

  4. 运行可执行程序出错！！！

    • 链接器：工作于链接状态，工作时需要 -l和-L
    • 动态链接器：工作于程序运行阶段，工作时需要提供动态库所在目录位置
    • 解决方法：export LD_LIBRARY_PATH=动态库路径 （关闭进程后失效）（建议绝对路径）

makefile

  • 一个规则
    • 目标：依赖条件
      （一个tab缩进）命令
      1. 目标的时间必须晚于依赖条件的时间没否则更新目录
      2. 依赖条件如果不存在，则寻找新的规则去依赖
        • ALL:指定makefile的终极目标
  • 2个函数
    • src = $(wildcard ./*.c) 匹配当前工作目录下的所有.c文件，将文件名组成列表赋值给变量src
    • obj = $(patsubst %.c,%.o,$(src)) 将参数3中包含参数1的部分替换为参数2
      clean:(没有依赖)
      -rm -rf $(obj) a.out （”-“的作用:删除不存在文件时，不报错，顺序执行结束）

  • 三个自动变量

    • $@: 在规则的命令中，表示规则中的目标
    • $<: 在规则的命令中，表示第一个依赖条件,如果将该变量用于模式规则中,它将条件依赖列表中的依赖依次取出,套用模式规则
    • $^: 在规则的命令中，表示所有依赖条件

  • 模式规则

    • %.o : %.c
      gcc -c $< -o $@

  • 伪目标

    .PHONY: clean ALL

进程间通信
管道

实现原理:内核借助环形队列机制,使用内核缓冲区实现
特质:

  1. 伪文件
  2. 管道中的数据只能一次读取
  3. 数据在管道中,只能单向流动

局限性:

  1. 自己写不能自己读
  2. 数据不可以反复读
  3. 半双工通信
  4. 血缘关系进程间可用

int pipe(int fd[2])

  • fd[0]:读端
  • fd[1]:写端

返回值:

  • 成功:0
  • 失败:1 errno

管道的读写行为

读管道:

  1. 管道有数据,read返回实际读到的字节数
  2. 管道无数据:
    1. 无写端:read返回0
    2. 有写端:read阻塞等待

写管道:

  1. 无读端: 异常终止(SIGPIPE)
  2. 有读端:
    1. 管道已满:阻塞等待
    2. 管道未满:返回写出的字节个数

    • event：
    • 返回值：成功0，失败-1

  • int epoll_wait(int epfd, struct epoll_event *events,int maxevents, int timeout);

    • epfd：根节点fd

      • -1：阻塞
      • 0：不阻塞
      • >0：超时时间
    • 返回值：
      • >=0：满足监听事件的总个数
      • -1：失败

  • ET和LT

    • 水平触发（LT）是只要读缓冲区有数据，就会一直触发可读信号
    • 边缘触发（ET）仅仅在空变为非空的时候通知一次
